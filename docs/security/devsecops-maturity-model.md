# DevSecOps Maturity Model (Template)

Use this to assess how mature a team's security practices are.

---

## Level 0 - Ad Hoc

- No security processes  
- No testing beyond manual  

---

## Level 1 - Basic

- Linting + simple tests  
- Secrets not in code  
- Basic dependency scanning  

---

## Level 2 - Intermediate

- CI with automated tests  
- SAST scanning  
- Threat modeling practiced  
- Formal review checklists  

---

## Level 3 - Advanced

- IAST/DAST tools  
- Automated security gates  
- Shift-left security fully adopted  
- API contract validation  

---

## Level 4 - Optimized

- Security champion program  
- Continuous risk assessment  
- Real-time monitoring + anomaly detection  
- Red/blue/purple team cycles
